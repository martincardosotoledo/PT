Algunas de las mejoras que se me fueron ocurriendo (seguramente hay muchas más, no anoté todo):

-paginación en comboboxes (por ej. clientes en la edición de envíos)
-el connection string nunca debería haber estado en el web.config
-el campo EstadoEnvio debería estar en su propia tabla, y usarse una FK desde la tabla Envios
-el campo ProveedorPaqueteria debería tener su propia tabla también probablemente
-hay muchas cosas que pulir en las clases helpers de nhibernate y las clases base de las entidades (entidadBase). Yo las simplifiqué para que simplemente funcionen.
Por ej. entidadBase podría tener un parametro genérico para el tipo del ID
-en la grilla de envíos se podría paginar en la base de datos en vez de la UI
-ActualizarEstado debe hacerse dentro de una transaccion
-se pueden optimizar las consultas de nhibernate para traer todos los datos a usar en un solo viaje
-agregar httpmodule para que nhibernate descarte los cambios a sesiones con cambios no comitteados
-poner cómo título de página el envío en edición, para lo cual debería haber un código de envio o similar
-centralizar textos como el "Elegir" de los combos
-en vez de traer toda la entidad como "Provincia" podría traerse una proyección pára los combos. Y así con el resto.
-manejo de errores centralizado, para capturar todos los errores no manejados y que nunca le llegue al usuario el error real
-no suelo usar los filtros automaticos de la grilla de devexpress. Ahora lo hice sólo por simplicidad. Siempre pongo los filtros fuera de la grilla para aplicarlos a la base de datos
-agregar logging, manejo  global de errores, manejo de concurrencia (por ej. no editar un envio que se elimino en otro lado)
-mostrar los errores de forma más bonita que en una lista hecha a mano. Me gustan los popups no intrusivos que desaparecen solos
-usar una clave unica para identificar cada página, para poder tener un session índependiente por instancia de página. Ahora sólo puede haber una por browser
-localizar devexpress al español
